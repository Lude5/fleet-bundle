Why this exists

The "Open in Google Analytics" buttons elsewhere just deep-link you to GA's own UI. With Data API credentials, the master admin can fetch sessions, users, page-views, bounce rate, top pages, top countries, device breakdown directly from GA and render them next to your click data — no tab-switching.

Step 1 · Create a service account

  1. Go to console.cloud.google.com and create a new project (or use an existing one). Name it whatever — "lude5-master" works.
  2. In the project, open Google Analytics Data API and click Enable.
  3. Go to IAM & Admin → Service Accounts → + Create Service Account. Name it "master-admin-ga". Skip the optional grants. Click Done.

Step 2 · Generate & save the key

  1. Click into the new service account → Keys tab → Add Key → Create new key → JSON. A file downloads.
  2. Save the downloaded JSON file as {{ '/'.join([request.host.split(':')[0], '_master_admin', '.ga-creds.json']) }} on your PC — i.e. drop it into C:\Users\talan\clients\_master_admin\.ga-creds.json.
  3. Copy the service account email address (looks like master-admin-ga@your-project.iam.gserviceaccount.com) — you need it for the next step.

Step 3 · Grant access in GA

  1. Open Google Analytics.
  2. Make sure you're on the property you want to pull data for (e.g. rep.tools).
  3. Admin (bottom-left gear) → Property Access Management → + (top right) → Add users.
  4. Paste the service account email from Step 2. Role: Viewer. Uncheck "Notify new users by email" (optional). Click Add.
  5. Repeat for each property (one for rep.tools, one for jake's site, etc.) — the service account needs to be granted on every property whose data you want to pull.

Step 4 · Add property IDs

Each GA property has a numeric Property ID (different from the G-XXXXXXXXXX Measurement ID). Find it: GA → Admin → Property Settings → Property details → Property ID.

Verify it works

Once steps 1–4 are done, the master admin must be restarted (so it picks up the credentials file). Then:

  1. Restart: stop the master admin and run python app.py again.
  2. Install the Python library: pip install google-analytics-data (or pip install -r requirements.txt).
  3. Open any site's config page and click Test live data fetch. You should see something like: ✓ Live · 1,234 sessions · 567 users · 8,910 page views · bounce 42%
  4. The Analytics page will now show a "Google Analytics · live" panel with real metrics for every configured site.

GA Data API quota is generous (500 requests per project per day). No risk of overage on a single-user admin.
